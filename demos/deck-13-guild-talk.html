The artifact is the demo —
HTML in the enterprise

What business users and engineers can really get out of M365 Copilot and GitHub Copilot by asking for one self-contained HTML file.

Vishal Shah · This deck is itself a single .html file — press →

The problem

AI made writing cheap.
Reading is now the bottleneck.

  • Copilot can draft a 9-page status update in seconds — and nobody reads page 4.
  • Long documents flatten structure: timelines, diffs, dashboards and decision tables all become paragraphs.
  • The result is familiar: we skim, we miss things, we ask for "a quick summary" of the summary.
The idea

Ask for one HTML file instead of a wall of text.

A single .html file can hold the timeline as a timeline, the diff as a diff, the dashboard as a dashboard — interactive, printable, and openable on any machine your company owns.

Everything in this talk is built on tools ABC Corp-like enterprises already run: M365 Copilot for business users, GitHub Copilot for engineers.

Why HTML — argument, not vendor claim

Six properties regulated enterprises happen to love

  • One file — self-contained, no dependencies to break or to security-review.
  • Opens everywhere — every locked-down laptop already has a browser.
  • No new licences — no extra tooling, viewers, or runtimes to procure.
  • Plain text under the hood — versionable and diffable in git like any code.
  • Attachable — drops into a change ticket, audit pack, or mail as-is.
  • Works air-gapped — zero external requests, by construction.
Lens 1 · M365 Copilot · business users

This is documented, not hoped for

Copilot Pages "lets you create, edit, and preview runnable code directly inside a page." Microsoft Support — "Build lightweight apps within Microsoft 365 Copilot Pages"
Code previews in Copilot Chat & Pages: Cloud Policy, enabled by default, admin-controllable. Microsoft Learn — Copilot Pages admin configuration

Universal fallback in any tenant: ask for one HTML file → copy → save as .html → open.

Lens 2 · GitHub Copilot · regulated engineering

The same move works under governance

Agent mode "will determine which files to make changes to, offer code changes and terminal commands for the user's approval, and iterate to remediate issues until the original task is complete." GitHub Docs — Copilot features

So alongside the code change, the agent can write the review pack, the ADR, the post-mortem, the evidence report — as files in the repo, under the same enterprise policies as everything else.

The honesty slide

What I am not claiming

  • The example artifacts were built with an AI coding agent to show the output format — they are not screenshots of Copilot output. Each one says so, on itself.
  • Every artifact ships its own prompt — reproduce it in your tenant and judge the result yourself.
  • Model output varies. The format's properties don't — and the platform capabilities cited are documented by Microsoft and GitHub.
  • All data is fictional: ABC Corp does not exist.
How to start on Monday

One experiment per lens

Business users

  • Take your next status update's raw notes.
  • Ask Copilot for a single-file HTML one-pager (steal the prompt from artifact #1).
  • Preview it in Pages — or save as .html and open it. Send that instead.

Engineers

  • On your next significant PR, ask agent mode for a review pack or ADR beside the change.
  • Commit the artifact to the repo — it diffs like code because it is code.
  • Attach it to the change record. Watch your auditors smile.
